PDFForm Class Reference

Inherits from NSObject
Declared in PDFForm.h

Overview

The PDFForm class represents a Widget Annotation owned by an interactive PDF form corresponding to a Field Dictionary contained in the ‘Fields’ array of the document’s ‘AcroForm’ dictionary. Thus each instance of PDFForm represents a unique rectangle on the PDF document where user interaction is permitted, whether through pressing or typing text. A ‘Field’ is a collection of PDFForm with the same name. All forms in a field have the same value. A ‘Field’ represents a coherent group of forms that work together to present and collect a common unified piece of information. For example a field may consist of the two button forms named ‘Sex’ and marked ‘Male’ and ‘Female’ respectively to collect the information of a person’s gender. A form can create a UIView representation of itself that can respond to user interaction.

PDFChoiceField* comboBox = [comboBoxTypeForm createWidgetAnnotationViewForSuperviewWithWidth:webView.bounds.size.width Margin:9.5]; [webView.scrollView addSubview comboBox]; [comboBox release];

exportValue

This is used with button forms only. Gives the name of the choice represented by that button.

@property (nonatomic, strong, readonly) NSString *exportValue

Discussion

If a button is part of a group of buttons where only one may be selected at once, then it’s value is the exportValue of the selected button. All buttons in such a group have the same name and represent a single form conceptually. Thus all forms in a radio button or check box group represent the field and always have the same value. A button is selected if and only if its exportValue is the same as its value. If no buttons are selected, or an unselected button is single and not part of a group, then the value is nil.

Returns a view to represent the form.

Parameters

vwidth

The width of the superview bounds.

xmargin

The horizonal margin width for the actual content frame in the superview.

ymargin

The vertical margin height for the actual content frame in the superview.

Return Value

A new view representing the form. The caller is responsible for releasing the new form view.

Discussion

The superview must be a UIScrollView or a subclass of UIScrollView.

initWithFieldDictionary:page:parent:

Creates a new instance of PDFForm based on a leaf AcroForm field dictionary and a PDF page

- (instancetype)initWithFieldDictionary:(PDFDictionary *)leaf page:(PDFPage *)pg parent:(PDFFormContainer *)p

Parameters

leaf

Either a terminal Acroform field dictionary (i.e. with no children), a terminal widget annotation dictionary, or a union of both. If leaf is a widget annotation, the parent element of the dictionary is used to traverse up the field tree to read inherited values from its parent field dictionary.

pg

The page that contains the form.

p

The parent.

Return Value

A new PDFForm object.
